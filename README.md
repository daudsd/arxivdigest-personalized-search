# arXivDigest Personalized Search

Dataset and evaluation scripts for the MSc thesis *Search Personalization for ArXivDigest* (University of Stavanger, 2026).

## Overview

This repository supports offline evaluation of personalized scientific paper search on arXivDigest. It contains user keyword profiles, two query sets, candidate documents retrieved by three rankers, relevance judgments generated by an LLM judge (Llama 3.3 70B), and all scripts needed to reproduce the experiments.

## Query Sets

Two query sets are used throughout:

- **Full query set**: all 969 CiteSeerX queries (`queries/trec_citeseerx_queries.csv`)
- **Personalized query set**: per-user subset filtered by `queries/query_profile_scores.jsonl` — keep only (user, query) pairs with score=2. Of 9,610 pairs (10 users × 961 queries), 845 (4.4%) pass this filter.

## File Formats

**candidates/*.jsonl** — one line per query:
```json
{"qid": "citeseerx-q1", "qstr": "ontology", "hits": [{"doc_id": "...", "score": 1.23}, ...]}
```

**candidates/candidates_personalized_*.jsonl** — one line per (query, user) pair:
```json
{"qid": "citeseerx-q1", "uid": "2", "qstr": "ontology", "keywords": ["information retrieval", ...], "hits": [...]}
```

**queries/query_profile_scores.jsonl** — one line per (user, query) pair:
```json
{"uid": "2", "qid": "citeseerx-q1", "qstr": "ontology", "score": 2}
```

**qrels/*.txt** — TREC format:
```
qid 0 doc_id relevance_score
```
For personalized qrels, `qid` is `{query_id}_u{user_id}` (e.g. `citeseerx-q1_u2`).

## Relevance Scale

All judgments use the standard TREC four-point scale:
- 0 = not relevant
- 1 = marginally relevant
- 2 = relevant
- 3 = highly relevant

## LLM Judge

Relevance judgments were generated by **Llama 3.3 70B** via the University of Stavanger's Ollama inference server. Three prompt variants were used.

### Non-personalized judge

Assesses relevance to the query alone. Used for `qrels_bm25`, `qrels_dense`, `qrels_hybrid`, and `qrels_pooled`.

```
Rate the relevance of the document to the query on a scale of 0 to 3.
0 = not relevant
1 = marginally relevant
2 = relevant
3 = highly relevant

Query: {query}
Title: {title}
Abstract: {abstract}

Reply with a single digit (0, 1, 2, or 3):
```

### User-aware judge

Assesses relevance to both the query and the user's keyword profile. Used for all `qrels_personalized_*` and `qrels_baseline_useraware_*`.

```
Given a user with research interests in: {keywords}

Rate the relevance of the document to their query on a scale of 0 to 3.
0 = not relevant
1 = marginally relevant
2 = relevant
3 = highly relevant

Query: {query}
Title: {title}
Abstract: {abstract}

Reply with a single digit (0, 1, 2, or 3):
```

### Query-profile likelihood

Scores how likely a user would issue a query given their research interests. Used to build the personalized query set.

```
Given a user with research interests in: {keywords}

How likely is it that this user would issue the following search query?
0 = unlikely (unrelated to their interests)
1 = possible (some overlap)
2 = very likely (directly relevant to their interests)

Query: {query}

Reply with a single digit (0, 1, or 2):
```

All prompts use greedy decoding (temperature 0) and ask for a single digit. The abstract is truncated to 800 characters.

## Quick Start (Evaluation Only)

If you only want to reproduce the evaluation results without re-running retrieval or judging:

```bash
# Non-personalized retrieval (pooled non-personalized judgments — primary evaluation)
python3 scripts/evaluate.py --mode bm25 dense hybrid \
  --data-dir qrels

# Personalized retrieval (personalized judgments)
python3 scripts/evaluate.py \
  --mode personalized_bm25 personalized_dense personalized_hybrid

# Fair comparison: all systems under personalized judgments
python3 scripts/evaluate.py \
  --mode baseline_useraware_bm25 baseline_useraware_dense baseline_useraware_hybrid \
         personalized_bm25 personalized_dense personalized_hybrid

# Personalized query set evaluation
python3 scripts/evaluate.py \
  --mode bm25 dense hybrid \
         personalized_bm25 personalized_dense personalized_hybrid \
  --filter-queries queries/query_profile_scores.jsonl

# Alpha sweep
python3 scripts/alpha_sweep.py --base bm25 dense hybrid --plot
```
