# arXivDigest Personalized Search Dataset

Dataset for the MSc thesis *Search Personalisation for ArXivDigest Platform* (University of Stavanger, 2026).

## Overview

This dataset supports offline evaluation of personalized scientific paper search. It consists of user keyword profiles, two query sets, candidate documents retrieved by three rankers, and relevance judgments generated by an LLM judge (Llama 3.3 70B).

## Structure

```
├── queries/
│   ├── trec_citeseerx_queries.csv       # 969 queries from TREC OpenSearch 2016 CiteSeerX track
│   └── query_profile_scores.jsonl       # per-user query likelihood scores (0/1/2)
├── profiles/
│   └── user_profiles.json               # 671 arXivDigest user keyword profiles
├── candidates/
│   ├── candidates_bm25.jsonl            # BM25 top-20 per query
│   ├── candidates_dense.jsonl           # Dense top-20 per query
│   ├── candidates_hybrid.jsonl          # Hybrid top-20 per query
│   ├── candidates_pooled.jsonl          # Pooled top-10 from all 3 rankers (up to 30/query)
│   ├── candidates_personalized_bm25.jsonl       # BM25 reranked per user (TF-IDF)
│   ├── candidates_personalized_dense.jsonl      # Dense reranked per user (TF-IDF)
│   ├── candidates_personalized_hybrid.jsonl     # Hybrid reranked per user (TF-IDF)
│   ├── candidates_personalized_bm25_emb.jsonl   # BM25 reranked per user (embedding)
│   ├── candidates_personalized_dense_emb.jsonl  # Dense reranked per user (embedding)
│   └── candidates_personalized_hybrid_emb.jsonl # Hybrid reranked per user (embedding)
└── qrels/
    ├── qrels_bm25.txt                   # Topical qrels for BM25 (per-ranker)
    ├── qrels_dense.txt                  # Topical qrels for Dense (per-ranker)
    ├── qrels_hybrid.txt                 # Topical qrels for Hybrid (per-ranker)
    ├── qrels_pooled.txt                 # Topical qrels for pooled candidates
    ├── qrels_personalized_bm25.txt      # User-aware qrels, TF-IDF reranked BM25
    ├── qrels_personalized_dense.txt     # User-aware qrels, TF-IDF reranked Dense
    ├── qrels_personalized_hybrid.txt    # User-aware qrels, TF-IDF reranked Hybrid
    ├── qrels_personalized_bm25_emb.txt  # User-aware qrels, embedding reranked BM25
    ├── qrels_personalized_dense_emb.txt # User-aware qrels, embedding reranked Dense
    ├── qrels_personalized_hybrid_emb.txt# User-aware qrels, embedding reranked Hybrid
    ├── qrels_baseline_useraware_bm25.txt   # User-aware qrels for BM25 (no reranking)
    ├── qrels_baseline_useraware_dense.txt  # User-aware qrels for Dense (no reranking)
    └── qrels_baseline_useraware_hybrid.txt # User-aware qrels for Hybrid (no reranking)
```

## Query Sets

Two query sets are used:

- **Full query set**: all 969 CiteSeerX queries (`trec_citeseerx_queries.csv`)
- **Personalized query set**: per-user subset filtered by `query_profile_scores.jsonl` (keep score=2 pairs only). Of 9,610 (user, query) pairs, 845 (4.4%) pass this filter.

## File Formats

**candidates/*.jsonl** — one line per query:
```json
{"qid": "citeseerx-q1", "qstr": "ontology", "hits": [{"doc_id": "...", "score": 1.23}, ...]}
```

**candidates/candidates_personalized_*.jsonl** — one line per (query, user) pair:
```json
{"qid": "citeseerx-q1", "uid": "2", "qstr": "ontology", "keywords": ["information retrieval", ...], "hits": [...]}
```

**query_profile_scores.jsonl** — one line per (user, query) pair:
```json
{"uid": "2", "qid": "citeseerx-q1", "qstr": "ontology", "score": 2}
```

**qrels/*.txt** — TREC format:
```
qid 0 doc_id relevance_score
```
For personalized qrels, `qid` is `{query_id}_u{user_id}` (e.g. `citeseerx-q1_u2`).

## Relevance Scale

All judgments use the standard TREC four-point scale:
- 0 = not relevant
- 1 = marginally relevant
- 2 = relevant
- 3 = highly relevant

## Judge Model

Relevance judgments were generated by **Llama 3.3 70B** via the University of Stavanger's Ollama inference server. Two prompt variants were used:
- **Topical judge**: scores relevance to the query alone
- **User-aware judge**: scores relevance to both the query and the user's keyword profile
